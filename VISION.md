Software is brittle. 
APIs change, DOM selectors break, and graphical interfaces are constantly redesigned. 
For decades, automation has relied on the underlying structure of applications—a fundamentally flawed approach when that structure is a moving target.

**ARIA was built to interact with software exactly as humans do: through vision.**

### The "Pure Vision" Philosophy

We believe that the future of computer automation is visual. 

If an application is accessible to a human operator, it must be accessible to an AI operator. ARIA does not parse HTML. It does not read accessibility trees. It looks at the screen, interprets the pixels, and moves the mouse. 

This philosophy unlocks universal compatibility:
- **Web Browsers**: ARIA operates without Chrome extensions or Playwright scripts.
- **Legacy Software**: ARIA operates mainframe terminals or 1990s accounting software seamlessly.
- **Video Games**: ARIA can navigate 3D environments just as easily as spreadsheets.

### The OS as the API

By treating the Operating System display as the ultimate API surface, we democratize automation. You no longer need developer APIs or Zapier integrations to connect two applications. ARIA acts as the universal bridge, typing and clicking across boundaries that traditional software cannot cross.

### Open Source, Local First

The most powerful digital employee should not be locked behind a corporate moat. ARIA is open-source, extensible, and capable of running fully locally using open-weight vision models (like Qwen2-VL or LLaVA). You own your automation layer.

Welcome to the visual era of computing.
